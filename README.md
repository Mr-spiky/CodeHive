# CodeHive / UIGen

A complete Next.js 14 (App Router) application that serves a dual purpose:
1. **CodeHive Landing Page (`/`)**: A premium, marketing and community homepage.
2. **UIGen Studio (`/studio`)**: An AI-powered user interface generation tool with an integrated code editor and live preview.

## 🚀 Tech Stack

- **Framework**: [Next.js 14](https://nextjs.org/) (App Router)
- **Styling**: [Tailwind CSS](https://tailwindcss.com/)
- **UI Components**: [shadcn/ui](https://ui.shadcn.com/)
- **Database ORM**: [Prisma](https://www.prisma.io/) (SQLite for development)
- **Authentication**: NextAuth.js
- **Icons**: Lucide React

---

## 📁 Project Structure

### Root Configuration Files
- `package.json` & `package-lock.json`: Project dependencies and NPM scripts (`dev`, `build`, `start`).
- `tailwind.config.ts` / `postcss.config.mjs`: Tailwind styling configuration.
- `next.config.ts`: Next.js core settings.
- `components.json`: Configuration for shadcn/ui.
- `prisma/schema.prisma`: Database schema defining `User` and `Project` models.
- `middleware.ts`: Next.js middleware used to protect authenticated routes (like `/studio`).

### 📂 `src/app/` — Routing & Pages
The `app` directory uses the Next.js App Router for all page definitions.
- `layout.tsx`: The root React layout. Injects global fonts, styles, and context providers.
- `globals.css`: Global base CSS, Tailwind imports, and custom utility classes.
- `page.tsx & main-content.tsx`: The primary CodeHive Landing page combining various sections.
- `studio/page.tsx`: The central entry point to the AI UI layout system.
- `[projectId]/page.tsx`: Isolated dynamic route serving a specific UIGen project payload.
- `api/chat/route.ts`: Streaming API route responsible for handling AI conversational logic.

---

### 📂 `src/components/` — The Building Blocks
All reusable React functional components. Organized by feature:

#### `/codehive/` - 🌐 Landing Page Elements
Everything rendered on the root `/` page.
- `Navbar.tsx`, `HeroSection.tsx`, `Footer.tsx`: Core page frame.
- `AboutSection.tsx`, `VisionMissionSection.tsx`, `TractionSection.tsx`: Community identity elements.
- `CommunityGroupsSection.tsx`, `CoreTeamSection.tsx`, `UpcomingEventSection.tsx`: Social and interactive showcases.
- `SponsorSection.tsx`: Marquee grid showcasing partner logic.

#### `/editor/` & `/preview/` & `/chat/` - 💻 UIGen Studio Features
Everything required for the interactive UI generator.
- `chat/`: Contains `ChatInterface.tsx`, `MessageList.tsx`, etc., which handle AI prompt messaging and rendering markdown.
- `editor/`: Contains `CodeEditor.tsx` (Monaco powered) and `FileTree.tsx`.
- `preview/`: Renders a live `<iframe>` sandbox generated from code (`PreviewFrame.tsx`).

#### `/auth/`
Modals and handlers.
- `AuthDialog.tsx` & `SignInForm.tsx` & `SignUpForm.tsx`

#### `/ui/`
These are foundational, un-opinionated UI primitives generated by **shadcn/ui** (Buttons, Dialogs, Inputs, Tabs, etc.).

---

### 📂 `src/lib/` — Business Logic & Tooling
Core utilities and logic layers bridging UI and server operations.

- `prisma.ts`: Initializes the singleton database client.
- `auth.ts`: Configuration rules and adapters for NextAuth.
- `provider.ts`: Manages connections to large language models (Anthropic/Gemini).
- `file-system.ts`: Powers the **virtual in-memory file system**, allowing the code editor tree to mirror project structures.
- `contexts/`: React context wrappers managing global state (`chat-context`, `file-system-context`).
- `transform/jsx-transformer.ts`: Powerful processor turning raw JSX/TSX into browser-runnable JS for live sandboxing.
- `tools/`: Server-executed functions that the AI can trigger (e.g., `str-replace.ts` for modifying file states).

### 📂 `src/actions/` — Server Actions
Next.js server-side executable actions, heavily database-dependent.
- `create-project.ts`: Generates a fresh UIGen setup.
- `get-projects.ts` / `get-project.ts`: Queries specific workloads based on user authentication.

## 🛠️ Getting Started

### 1. Install Dependencies
```bash
npm install
```

### 2. Database Setup
```bash
npx prisma generate
npx prisma db push
```

### 3. Run Development Server
```bash
npm run dev
```
Navigate to `http://localhost:3000` to view the CodeHive landing page, or `/studio` to open the UIGen engine.
